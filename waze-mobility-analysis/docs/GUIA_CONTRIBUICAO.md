# Guia de Contribuição

Obrigado por considerar contribuir para o projeto de **Análise de Mobilidade Urbana com Dados Waze**! Este guia fornece diretrizes para colaboradores.

## Como Contribuir

### 1. Reportar Bugs

Se você encontrar um bug, por favor abra uma **issue** no GitHub com as seguintes informações:

*   Descrição clara do problema
*   Passos para reproduzir o bug
*   Comportamento esperado vs. comportamento observado
*   Versão do R e dos pacotes utilizados
*   Mensagens de erro (se houver)

### 2. Sugerir Melhorias

Sugestões de novas funcionalidades ou melhorias são bem-vindas! Abra uma **issue** descrevendo:

*   O problema que a melhoria resolve
*   A solução proposta
*   Exemplos de uso (se aplicável)

### 3. Enviar Pull Requests

Para contribuir com código:

1.  **Fork** o repositório
2.  Crie uma **branch** para sua feature: `git checkout -b feature/minha-feature`
3.  Faça suas alterações e **commit**: `git commit -m "Adiciona minha feature"`
4.  **Push** para o GitHub: `git push origin feature/minha-feature`
5.  Abra um **Pull Request** descrevendo suas alterações

### Padrões de Código

*   Utilize **nomes descritivos** para variáveis e funções
*   Adicione **comentários** para explicar lógica complexa
*   Siga o **estilo tidyverse** para código R
*   Teste suas alterações antes de enviar

### Documentação

*   Atualize o `README.md` se necessário
*   Adicione documentação para novas funções
*   Inclua exemplos de uso

## Código de Conduta

Este projeto adota um código de conduta baseado em respeito e colaboração. Esperamos que todos os colaboradores:

*   Sejam respeitosos e inclusivos
*   Aceitem críticas construtivas
*   Foquem no que é melhor para a comunidade

## Contato

Para dúvidas ou sugestões, entre em contato através das **issues** do GitHub.
